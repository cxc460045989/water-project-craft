## 2. 上行数据帧（仪器 → 计算机）

仪器自动每 **1 秒** 发送一次。

### 2.1 帧格式

```
S + 炉膛温度(4位) + 天平数据(7位) + 联机标志(1位) + 仪器按键(1位) + END
```

- 总长度: **17 字节**（ASCII 文本帧）
- 所有字段均为十进制数字字符

### 2.2 帧结构详解

| 位置 | 长度 | 字段名 | 说明 |
|------|------|--------|------|
| 0 | 1 | 起始标志 | 固定 `S` |
| 1-4 | 4 | 炉膛温度 | 原始值 / 10 = 实际℃ |
| 5-11 | 7 | 天平数据 | (原始值 − 3000000) / 10000 = 实际 g |
| 12 | 1 | 联机标志 | 0/1 |
| 13 | 1 | 仪器按键 | 1 = 有按键按下（功能等同称量样重时的确认按键） |
| 14-16 | 3 | 结束标志 | 固定 `END` |

### 2.3 示例解析

```
原始帧: S0850301001701END
         |  |       |||
         S  0850    3010017 0 1 END
         起始 温度   天平   联机 按键 结束
```

- **温度**: 0850 / 10 = **85.0℃**
- **天平数据**: (3010017 − 3000000) / 10000 = **1.0017 g**
- **联机标志**: 0
- **仪器按键**: 1（有按键）

### 2.4 解析规则

1. 先判断起始标志 `S` 和结束标志 `END` 是否正常
2. 正常则解析数据，否则丢弃
3. 帧长 ≠ 17 或字段含非数字字符 → 丢弃

## 3. 下行指令集（计算机 → 仪器）

所有指令为 **十六进制字节**，帧头 `0x5A`，帧尾 `0x44`。

### 3.1 固定 4 字节指令（5A 4D XX 44）

| 编号 | 指令名称 | 功能码 | 完整指令 (hex) |
|------|----------|--------|----------------|
| 1 | 握手 | `01` | `5A 4D 01 44` |
| 2 | 蜂鸣器响1秒 | `07` | `5A 4D 07 44` |
| 3 | 样盘不停转动 | `13` | `5A 4D 13 44` |
| 4 | 样盘上升到高位 | `14` | `5A 4D 14 44` |
| 5 | 样盘降到低位 | `15` | `5A 4D 15 44` |
| 6 | 天平清零（去皮） | `16` | `5A 4D 16 44` |
| 7 | 天平校正 | `17` | `5A 4D 17 44` |
| 8 | 关闭炉盖 | `18` | `5A 4D 18 44` |
| 9 | 打开炉盖 | `19` | `5A 4D 19 44` |
| 10 | 仪器称量样重状态 | `12` | `5A 4D 12 44` |
| 11 | 解除称重状态 | `11` | `5A 4D 11 44` |
| 12 | 关闭鼓风、氮气、加热 | `10` | `5A 4D 10 44` |
| 13 | 关闭加热 | `1B` | `5A 4D 1B 44` |
| 14 | 开鼓风 | `1C` | `5A 4D 1C 44` |
| 15 | 关鼓风 | `1D` | `5A 4D 1D 44` |
| 16 | 开氮气 | `1E` | `5A 4D 1E 44` |
| 17 | 关氮气 | `1F` | `5A 4D 1F 44` |
| 18 | 仪器复位 | `20` | `5A 4D 20 44` |
| 19 | 开蜂鸣（持续） | `21` | `5A 4D 21 44` |
| 20 | 关蜂鸣 | `22` | `5A 4D 22 44` |
| 21 | 样盘移动一位 | `29` | `5A 4D 29 44` |
| 22 | 样盘移动到1号位 | `30` | `5A 4D 30 44` |
| 23 | 关闭鼓风、氮气、氧气 | `32` | `5A 4D 32 44` |
| 24 | 水分开始测试（开鼓风） | `33` | `5A 4D 33 44` |
| 25 | 水分开始测试（开氮气） | `34` | `5A 4D 34 44` |
| 26 | 开氧气 | `0E` | `5A 4D 0E 44` |
| 27 | 关氧气 | `0F` | `5A 4D 0F 44` |

### 3.2 移动样位指令（4 字节）

```
5A 4D XX 44
```

**编码规则**: `XX = 0x34 + 样位号`

| 示例 | 指令 (hex) |
|------|------------|
| 移动到 1 号位 | `5A 4D 35 44` |
| 移动到 9 号位 | `5A 4D 3D 44` |

> 样位号范围: 1~99

### 3.3 开始控温指令（7 字节）

```
5A 57 x1 x2 x3 x4 44
```

**编码规则**: 目标温度（℃）转为 4 位十进制，每位编码为 **数值字节** (0x00~0x09)

| 示例 | 指令 (hex) | 说明 |
|------|------------|------|
| 控温 105℃ | `5A 57 00 01 00 05 44` | 105 → "0105" → bytes(0,1,0,5) |

> ⚠️ 注意: 温度参数是**数值字节**，不是 ASCII 编码。  
> 105℃ → `bytes([0, 1, 0, 5])` ✅  
> 105℃ → `b"0105"` = `bytes([0x30, 0x31, 0x30, 0x35])` ❌

### 3.4 发送天平数据到仪器（11 字节）

```
5A 58 x1 x2 x3 x4 x5 x6 x7 x8 44
```

**编码规则**:
1. 计算中间值: `mid = weight_g × 10000 + 1000000`
2. 将 mid 转为 8 位十进制，每位编码为**数值字节**

| 示例 | 指令 (hex) |
|------|------------|
| 发送 1.0019 g | `5A 58 00 01 00 01 00 00 01 09 44` |

> 验证: 1.0019 × 10000 + 1000000 = 1010019 → "01010019" → bytes(0,1,0,1,0,0,1,9)

## 4. 指令响应帧（仪器 → 计算机）

仪器收到下行指令后，通过上行通道返回响应帧。

### 4.1 通用响应格式

```
4F 4B XX 45 4E 44
```

即 ASCII 字符串 `OK` + 功能码字节 + `END`

| 字段 | 长度 | 说明 |
|------|------|------|
| 4F 4B | 2 | ASCII `OK` |
| XX | 1 | 被响应的功能码字节 |
| 45 4E 44 | 3 | ASCII `END` |

### 4.2 控温指令响应（特例）

发送控温指令 `5A 57 x1 x2 x3 x4 44` 后，返回:

```
4F 4B x1 x2 x3 4E 44
```

仪器回显 x1, x2, x3 三个参数字节（共 7 字节）。

### 4.3 响应处理规则

1. 发送下行指令后，应先检测是否有新的上行响应帧
2. 收到 `OK...END` 响应 → 指令执行成功
3. 超时无响应 → 重发指令

## 5. 数值编码规则汇总

| 数据类型 | 编码方式 | 示例 |
|----------|----------|------|
| 上行温度 | ASCII 十进制字符，4位 | `"0850"` → 85.0℃ |
| 上行天平 | ASCII 十进制字符，7位 | `"3010017"` → 1.0017g |
| 上行联机/按键 | ASCII 单字符 | `"0"` / `"1"` |
| 下行功能码 | 单字节 hex | `0x16` → 天平清零 |
| 下行温度参数 | 数值字节（逐位，0x00~0x09） | 105℃ → `[0x00,0x01,0x00,0x05]` |
| 下行天平参数 | 数值字节（逐位，0x00~0x09） | 1.0019g → `[0x00,0x01,0x00,0x01,0x00,0x00,0x01,0x09]` |
| 响应帧 | ASCII 混合 hex | `OK` + 参数字节 + `END` |

## 6. 开发注意事项

1. **数值字节 vs ASCII**: 下行变长指令中的数字参数是数值字节 (0x00~0x09)，**不是** ASCII 数字字符 (0x30~0x39)。这是最常见的错误源。

2. **帧边界**: 上行帧固定 17 字节，下行响应帧长度可变（6~7 字节）。串口读取时需处理粘包/半包。

3. **握手指令已废弃**: `5A 4D 01 44` 不得在新代码中使用，统一使用 `send_cmd_with_uplink_check` 发送指令并等待上行帧确认。

4. **上行心跳**: 仪器每 1 秒自动发送上行帧，与下行指令的响应帧共享同一上行通道。需要区分心跳帧和响应帧。
